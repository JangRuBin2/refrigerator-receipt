# 공유하기 링크 개선 (2026-03-18)

## 변경 배경

FightMe 프로젝트에서 토스 공유 링크 구현 중 발견된 문제를 반영하여 밀키퍼에도 동일 패턴을 적용.

### 기존 문제

1. **딥링크 쿼리 파라미터 충돌**: `intoss://acorn/ko/recipe?id=xxx` 형태의 딥링크에서 Toss 플랫폼이 `_deploymentId`를 쿼리 파라미터로 추가하면서 URL이 꼬임 → 페이지 로드 실패 (`ERR_HTTP_RESPONSE_CODE_FAILURE`)
2. **OG 이미지 없음**: 공유 시 미리보기 이미지가 표시되지 않음
3. **공유 메시지에 설명 없음**: 링크만 전송되어 맥락 부족
4. **환경별 분기가 호출부에 노출**: 토스/브라우저 분기가 컴포넌트에 직접 작성됨

## 변경 내용

### 1. `src/lib/tossShare.ts` - 전면 개편

**Before:**
```typescript
const deepLink = `intoss://acorn/${locale}/recipe?id=${recipeId}`;
const tossLink = await getTossShareLink(deepLink);
await share({ message: tossLink });
```

**After:**
```typescript
// path param 방식 (쿼리 파라미터 충돌 방지)
const deepLink = `intoss://acorn/${locale}/recipe/${recipeId}`;
// OG 이미지 포함
const tossLink = await getTossShareLink(deepLink, OG_IMAGE_URL);
// 설명 텍스트 + 링크
const message = `${recipeTitle} - 밀키퍼 AI 레시피를 확인해보세요!\n${tossLink}`;
await share({ message });
```

**추가된 기능:**
- OG 이미지 URL 전달 (Supabase Storage `og-image/share.png`)
- 레시피 제목 포함 공유 메시지
- 토스 → Web Share API → clipboard 순차 fallback (함수 내부 처리)
- 반환값 `'shared' | 'copied' | 'failed'`로 통일

### 2. `src/features/recommend/AiRecipeMode.tsx` - 호출 단순화

**Before:**
```typescript
if (isTossEnvironment()) {
  // 토스 분기
  await shareTossRecipeLink(recipeId, locale);
} else {
  // 브라우저 분기
  const file = await renderRecipeToImage(...);
  await shareImage(file, recipe.title);
}
```

**After:**
```typescript
const result = await shareTossRecipeLink(recipeId, locale, recipe.title);
if (result === 'copied') toast.success(t('share.linkCopied'));
// 실패 시 이미지 공유 fallback
```

### 3. `src/app/[locale]/recipe/components/RecipeContent.tsx` - 경로 호환

**추가:** `/locale/recipe/{id}` path param 방식 지원
- 기존: `/ko/recipe?id=xxx` (쿼리)
- 추가: `/ko/recipe/xxx` (path param, 토스 딥링크)
- `usePathname` + `useMemo`로 양쪽 모두 파싱

## OG 이미지 설정

1. Supabase Dashboard → Storage → `og-image` 버킷 생성 (Public)
2. `share.png` (1200x600px) 업로드
3. URL: `{SUPABASE_URL}/storage/v1/object/public/og-image/share.png`

## 참고

- `intoss://` 스킴은 앱 정식 출시 후에만 동작
- 출시 전 테스트는 `intoss-private://appsintoss?_deploymentId=XXX` 사용
- `getTossShareLink` 함수가 자동으로 테스트/프로덕션 스킴 처리
